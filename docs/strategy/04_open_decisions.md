# Журнал решений и гипотез

Часть Strategy v2. Формат записи: статус / варианты / текущее рабочее
решение / какие данные нужны / когда вернуться / итоговое решение и дата.
Новые идеи после v2 попадают сюда с категорией: **принято / гипотеза /
поздний резерв** — и только решение владельца или данные переводят их в
стратегию.

## Р-1. Публичность связи UNIXOUND с проектом

- **Статус:** ✅ РЕШЕНО 2026-07-15 (владелец подтвердил)
- **Решение:** публично — «Проект развивается при поддержке студии UNIXOUND
  и других партнёров музыкального сообщества». Статус «партнёр-основатель» —
  внутри партнёрских материалов (честен, пока студия реально даёт ресурсы:
  часы, призы, продвижение). Не публикуются: владение, авторство кода,
  административная роль владельца.
- **Инварианты:** собственные показы маркируются; без преимущества в
  органике; учёт как кампании с расчётной стоимостью; частота ограничена;
  партнёрам доступны сопоставимые форматы.

## Р-2. Входное сообщение для каналов привлечения

- **Когда вернуться:** перед первой кампанией привлечения / договорённостью
  с релокантским каналом.

## Р-11. Раздел «Релизы» в меню бота

- **Статус:** 🔶 ФОРМАТ СОГЛАСОВЫВАЕТСЯ (2026-07-16). Владелец решил:
  разделу быть (не гипотеза). Формат согласован владельцем + Claude +
  ChatGPT, ниже. Код — по «да» владельца после трёх последних вопросов.
- **Зачем:** новый контент в боте (медиа), музыканты показывают работы
  (сообщество), автор сам делится карточкой — реклама бота; честная
  витрина студий через поле «где записано»; рубрика для дайджеста.

**Согласованный формат (MVP):**
- Меню: 🎵 Релизы → Новые релизы / Добавить релиз / Мои релизы.
- Карточка: обложка (обязательно), исполнитель, название, тип
  (сингл/EP/альбом/клип/live), дата, жанр (необязательно), описание
  (необязательно), «где записано» (необязательно), ссылки на площадки —
  несколько подписанных, хотя бы одна обязательна (YouTube, Spotify,
  Яндекс, Apple, Bandcamp, SoundCloud, VK, другая).
- Лента по свежести, без городов и без рубрик по жанрам (пока база мала).
- Архитектура: общая таблица listing (type='release') + своя мета-таблица
  по образцу Афиши (events_meta). Без 30-дневного срока, продлений,
  карантина и «продано» — свои статусы: опубликован / скрыт / удалён
  (+ «на модерации/отклонён», если владелец выберет премодерацию).
- Публикуем только официальное: синглы, EP, альбомы, клипы, live-релизы.
  Демки, «оцените сведение» — не сюда (позже отдельная рубрика
  «Творчество»). Галочка автора: «я автор/представитель или имею
  разрешение». Файлы не храним — только обложка и внешние ссылки.
- Бесплатно; без рейтингов, голосований, комментариев, чартов, плеера.
  Монетизация вокруг релизов — только позже, по общим правилам.
- Релизы не считаются в лимит 5 бесплатных объявлений.

**Осталось решить владельцу:** (1) название пункта меню («🎵 Релизы»?);
(2) модерация: сразу видно + уведомление админу с кнопкой «Скрыть»
(рекомендация Claude) или строгая премодерация (рекомендация ChatGPT) —
заложить оба режима за выключателем; (3) нужна ли кнопка связи с автором
в карточке (обе модели склоняются: на старте — нет).

## Р-4. Граница бесплатной и расширенной статистики

- **Статус:** 🔶 рабочее решение
- **Решение:** бесплатно — количество просмотров; платно — воронка
  (открытия, избранное, контакты), источники, динамика по дням, сравнение
  с категорией.
- **Когда вернуться:** перед этапом 4 (разовые продукты).

## Р-5. Лимиты бесплатного уровня

- **Статус:** 🔶 рабочее решение — 5 активных / 30 дней / напоминание
  ~25-й день / карантин.
- **Данные:** доля пользователей, упирающихся в лимит; доля продлений;
  жалобы. **Когда вернуться:** после 2–3 месяцев реальной статистики.

## Р-6. Частота показов собственных кампаний (UNIXOUND, антикафе)

- **Статус:** 🔶 рабочее решение — ограничение частоты обязательно, число
  подобрать при внедрении ротации (доля инвентаря собственных кампаний —
  видна из расчётной стоимости).

## Р-7. Способ оплаты B2B-кампаний

- **Статус:** ❓ ОТКРЫТО — счёт/перевод против Tribute.
- **Данные:** чек-лист проверки Tribute (правила Telegram, выплаты в
  Сербии, комиссии, налоги, возвраты). **Когда вернуться:** этап ручных
  продаж (первый платящий партнёр).

## Р-8. Цены всех продуктов

- **Статус:** ❓ ОТКРЫТО — до ручных продаж не назначать. Ориентир для
  партнёрки: стоимость эквивалентного охвата в Instagram.

## Р-9. Налоги и юридическое оформление платежей

- **Статус:** ❓ ОТКРЫТО — фактический блокер карточных платежей, важнее

## Журнал гипотез

| Гипотеза | Категория | Проверка |
|---|---|---|
| Донаты как репетиция платёжного контура | гипотеза | включить кнопку, смотреть техпрохождение, не доход |
| Антикафе как рекомендуемое место встречи | гипотеза (пилот) | объявить, считать использования |
| Консьерж (ручное сведение) | гипотеза (пилот, лимиты) | несколько случаев/нед, доля успешных |
| Призы свободными часами студии | гипотеза (пилот) | розыгрыш с объективными условиями |
| Добровольные двуязычные объявления (SR) | гипотеза | доля авторов, добавивших перевод |
| Вопрос «Удалось ли?» | принято в стратегию (слой 1) | доля ответов, распределение |
| Дайджест | принято в стратегию (слой 1) | открытия, возвраты после выпуска |
| Спрос-объявления `intent=request` | принято в стратегию (слой 1) | доля «ищу», их контактность |
| Переносимый профиль между странами | поздний резерв | — |
| Локальные хабы в других городах | поздний резерв | — |
