# Слой 2. Монетизация

Часть Strategy v2 (см. `00_strategy_map.md`). Заменяет коммерческую часть
`../monetization_plan.md` (архив).

## 1. Что НЕ монетизируется (зафиксировано)

- Контакты и связь с автором — бесплатны навсегда.
- Комиссия со сделок — нет (бот их не контролирует).
- Услуги UNIXOUND — вне платёжного контура бота: воронка заканчивается на
  «Связаться», дальше лично. Никаких оплат/бронирования/кабинета студии.
- Значок доверия не продаётся: платной может быть процедура рассмотрения,
  результат зависит от реальной проверки.
- Скрытая реклама; нерелевантные платные результаты; рассылки без согласия;
  искусственно бесполезный free; обещания гарантированных клиентов;
  продажа персональных данных.

## 2. Кто платит

1. **Обычный пользователь** — разовые покупки под конкретную нужду.
2. **Профессионал** (преподаватель, звукорежиссёр, мастер, постоянный
   продавец) — будущий Pro.
3. **Организация** (студия, школа, магазин, клуб, репбаза) — будущий
   Business-профиль (отдельная сущность; до реальных требований — вручную).
4. **Рекламодатель/партнёр** — кампании с доступом к аудитории.
5. **Работодатель/организатор со сроком** — срочные продукты (готовность
   платить выше: практическая ценность + дедлайн).

## 3. Продуктовая модель

**Free / разовые продукты / Pro / Business / партнёрские кампании.**
VIP, TOP, «Срочно» — способы продвижения объявления, не тарифы.

**Free** остаётся полноценным: просмотр, поиск, контакты, стандартные
объявления, базовая статистика, базовое избранное и один сохранённый поиск,
«сообщить, когда появится». Рабочие лимиты (проверить на данных): 5 активных
объявлений, 30 дней, напоминание ~25-й день, карантин после. Лимиты — для
актуальности базы, не для принуждения.

**Разовые продукты** (появляются на этапе 4–5, сначала вручную): поднятие
(без нарушения релевантности, с ограничением частоты и сроками); срочное
размещение (метка, 48–72 ч, блок срочных, уведомление подходящим);
продвижение вакансии/события; дополнительное объявление сверх лимита;
длительное размещение/автопродление; расширенная статистика. **Адресное
уведомление — компонент** (входит в срочное, продаётся и отдельно),
вырастает из бесплатного «сообщить, когда появится»; платными становятся
мгновенность, несколько запросов, города, приоритет.

**Pro** — после появления когорты регулярных профи; состав не проектировать
заранее. **Business** — после организаций, плативших вручную повторно.

Терминология статистики: «контактное действие», «переход к контакту» —
не «клиент» и не «сделка».

## 4. Гейты и операционная стоимость

Готовность автоматизировать продукт — четыре условия одновременно:

```
достаточный масштаб
× интенсивность намерения
× повторяемость (возвраты, повторные размещения, повторные покупки)
× приемлемая операционная стоимость
```

Первая покупка — любопытство; повторная — ценность. Экономика продукта:

```
Доход − прямые расходы − время владельца − модерация − обслуживание
```

Ручная продажа всегда раньше автоматизации: несколько раз продали руками →
автоматизируем. Успешность ручного теста измеряется заранее оговорённым
результатом для покупателя (показы/контакты), не фактом оплаты.

## 5. Партнёрские кампании

Сущность кампании: партнёр, текст/карточка RU/EN, место показа, раздел,
город, категория, даты, вес, частота на человека, активность; показы и
клики — события в `analytics_events`; переходы через `/go/`-редирект.

- **Форматы**: партнёр раздела, информационный партнёр, спонсор
  подборки/выпуска дайджеста, партнёр мероприятия, рекомендуемый сервис.
- **Прозрачность**: маркировка («Партнёр», «Реклама», «Рекомендуем партнёра
  сообщества»); оплаченное не выдаётся за органическое; допустимы отдельные
  спонсорские/срочные блоки и продвижение только среди релевантных.
- **UNIXOUND**: обычная кампания с расчётной стоимостью инвентаря — чтобы
  знать, сколько показов заняла, их эффективность, цену такого объёма для
  внешнего партнёра, и не съедает ли собственная кампания слишком много.
  Частота собственных показов ограничивается. Антикафе — отдельный участник
  ротации.
- **Промокоды партнёров** («скидка по коду из бота») + уникальная ссылка +
  вопрос «откуда узнали» — измерение офлайн-конверсии без платежей.
  Для UNIXOUND промокод не обязателен (контакт личный).
- **Партнёры- и профи-основатели**: первым — Business/Pro бесплатно на
  6–12 месяцев + бессрочный статус-бейдж, в обмен на наполнение и обратную
  связь. Не «бесплатно навсегда».
- Ценообразование: ориентир — стоимость эквивалентного охвата в
  Instagram-рекламе; CTR-гипотезы для калибровки: строка в меню 0,5–2%,
  карточка в разделе 2–5%.

## 6. Платёжная стратегия

- Разделять: продукт / место покупки / способ оплаты / посредник / способ
  получения денег / выданное право.
- **Цифровые функции внутри бота — Telegram Stars** (правила Telegram).
  Продавать конкретный результат, не «звёзды»; не строить доход на
  микроплатежах; конверсию тестировать отдельно.
- **B2B (кампании организаций) — вне бота**: счёт, карта, перевод; Tribute —
  кандидат (перед применением проверить правила Telegram, страны, выплаты в
  Сербии, комиссии, налоги, категории, возвраты).
- **Донаты — репетиция платёжного контура** (этап «простые эксперименты»):
  счёт → оплата → событие → запись → уведомление → отчётность, без выдачи
  прав. Продуктовых обязательств нет, но платёжная ответственность есть
  (учёт, поддержка, возврат, налоги). Не источник дохода, не приоритет.
- Архитектура независима от провайдера: заказ, продукт, цена, валюта,
  провайдер, попытка, статус, возврат, право, срок — отдельно. Таблицы
  `orders`/`payments`/`entitlements` НЕ создаются до подтверждённого
  платного продукта (этап 7).
- Модерация и антиспам (жалобы, блокировки, история нарушений) — раньше
  любых платежей: продвигать объявления на площадке с сомнительным
  содержанием нельзя.

## 7. Сценарные предположения (НЕ прогноз)

Гипотезы при допущении «потолок MAU Сербии за год 300–1000» — только для
управления ожиданиями, в решения не подставлять: пессимистично — прямой
доход ~0 (ценность: клиенты UNIXOUND + данные); базово — 50–150 €/мес к
9–12 мес; оптимистично — 300–500 €/мес при 2–3 Business. Один студийный
клиент из бота может стоить больше месячной выручки бота — учитывать в
оценке отдачи (см. `03_platform_and_scaling.md`, пять видов отдачи).
