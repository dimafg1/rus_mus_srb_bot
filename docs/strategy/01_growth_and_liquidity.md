# Слой 1. Рост аудитории и ликвидность

Часть Strategy v2 (см. `00_strategy_map.md`). До коммерческих этапов всё в
этом документе — и есть монетизационная работа, хотя денег не приносит.

## 1. Центральный цикл развития

```
Увидели спрос
→ нашли недостающее предложение
→ соединили людей
→ спросили, получилось ли
→ улучшили систему
→ рассказали об успешном результате
→ привлекли новых участников
```

Этот цикл одновременно создаёт аудиторию, доверие, данные, будущие платные
продукты и основу масштабирования.

## 2. Граф жизненных ситуаций (не линейная воронка)

Разделы бота обслуживают разные потребности одного человека. Движение — не
по линии, а по графу ситуаций:

```
Переезд                      Создание проекта
├── найти сообщество         ├── собрать состав
├── найти работу             ├── найти репетиционную базу
├── найти музыкантов         ├── найти специалиста
├── купить оборудование      ├── записать материал
└── найти площадку           └── организовать выступление
```

**Принцип «следующий полезный шаг»:** бот предлагает логичное продолжение
без агрессивной рекламы (разместил «ищу группу» → подборка репбаз и джемов;
опубликовал концерт → напоминание о фотографах/звукорежиссёрах; собрал
состав → подборка студий, включая UNIXOUND в общей прозрачной подборке).

Граница: бот развивает музыкальную активность сообщества в целом; рост
числа проектов естественно увеличивает спрос на студийные услуги, в т.ч.
UNIXOUND — но бот не ведёт каждого пользователя именно в неё.

## 3. Сторона спроса

У каждого рынка две стороны — универсальное свойство объявления
`intent = offer | request`, раздел определяет предмет:

```
Продаю инструмент / Куплю инструмент
Предлагаю услугу / Ищу услугу
Ищу работу / Ищу исполнителя
Предлагаю участие / Ищу участника
```

Объявление «ищу» — зафиксированная неудовлетворённая потребность. Из неё
органически вырастают: «сообщить, когда появится» (бесплатная базовая
функция — будущая основа адресных уведомлений), сохранённый поиск,
консьерж, радар дефицита, срочные продукты.

## 4. Возвраты пользователей

Бот без механизма возвращения — база, куда зашли один раз и забыли.

- **Дайджест** (еженедельный/периодический): новые музыканты и услуги,
  актуальные вакансии, события выходных, интересное с барахолки, «кого
  сейчас ищут». Даёт возвраты, просмотры объявлениям, место для спонсора
  подборки, витрину закрытых дефицитов.
- **Уведомления**: по сохранённому запросу, «появилось подходящее», напоминание
  о продлении. Всегда с возможностью отказаться; частота ограничена.
- **Избранное и сохранённый поиск** — базовые версии бесплатны.

## 5. Радар дефицита = рабочая очередь развития

Запрос без результата → задача администратора:

```
14 поисков «преподаватель вокала», результатов 0
→ найти 3–5 преподавателей в городе → лично пригласить
→ помочь с первыми карточками → уведомить искавших
```

Приоритизация: частота спроса × доля без результатов × вероятность найти
поставщика × ценность результата (5 срочных поисков звукорежиссёра важнее
20 расплывчатых запросов).

Каждый закрытый дефицит — контент дайджеста: «Вы часто искали преподавателей
вокала — в каталоге три новых специалиста».

## 6. Измеримость (первый технический этап)

Порядок внедрения (утверждён):

1. **Источники первого входа** — deep-link параметр `/start` сохраняется с
   первого дня (потерянный источник не восстановить): unixound, антикафе,
   канал, Instagram, мероприятие, партнёр, приглашение пользователя.
2. **Словарь аналитических событий** (`analytics_events`) — смысл раньше
   таблицы. Стартовый словарь: `user_started`, `listing_created`,
   `search_performed`, `search_no_results`, `listing_shown`,
   `listing_opened`, `contact_clicked`, `listing_extended`,
   `partner_shown`, `partner_opened`. Показы/клики кампаний живут здесь же,
   отдельные таблицы — когда заставит объём.
3. **Feature flags** — включение функций админу / группе / разделу /
   постепенно всем. Всё новое пишется «под выключателем».
4. **Кампании** — когда появится реальное место показа (UNIXOUND, антикафе,
   первый партнёр). Модуль — см. `02_monetization.md`.

Первая техническая цель — не коммерция, а измеримость поведения.

## 7. Метрики здоровья

**По ячейкам, не по больнице в среднем:** проект × город × раздел ×
категория. У каждой ячейки своя ликвидность: предложений, поисков,
открытий, контактов, запросов без результата, время до первого обращения.

Ключевые метрики (еженедельно):

- **Ликвидность: доля объявлений с ≥1 контактным действием за 14 дней —
  главная метрика проекта**;
- WAU; возвраты через 7/30 дней; новые объявления в неделю по разделам;
- доля поисков без результатов; доля продлений на 25-й день;
- **Time to First Value** (до первого полезного события: нашёл релевантное /
  опубликовал / сохранил поиск) — качество интерфейса;
- **Time to First Outcome** (до контакта/отклика/подтверждённого
  результата) — жизнеспособность рынка. Считать раздельно по намерениям:
  ищущий / автор / организация.

## 8. Вопрос «Удалось ли?»

Через несколько дней после контактного действия — один добровольный вопрос:
«Удалось получить полезный результат?» (Да, договорились / Связались, но не
договорились / Не получил ответа / Ещё в процессе / Не хочу отвечать).

Даёт: настоящую ликвидность (не «нажал», а «получилось»), качество
объявлений, выявление молчащих авторов, доказательства пользы для партнёров.

Ограничения: добровольно; не часто; без публичных рейтингов; не наказывать
по одному одностороннему ответу; не раскрывать ответ второй стороне; не
считать доказательством денежной сделки. На первом этапе — только
агрегированно; мягкая репутация («обычно отвечает») — отдельный поздний этап.

## 9. Привлечение и мобильность аудитории

Отток встроен в релокантскую аудиторию (уезжают дальше, возвращаются) —
постоянный вход новых важнее удержания текущих. Каналы: сообщества
«первых дней в Сербии», русскоязычные группы городов, школы языка, сервисы
переезда, группы жилья, музыкальные школы и магазины, мероприятия.

Входное сообщение для каналов — открытое решение владельца
(см. `04_open_decisions.md`).

Распродажа перед переездом — реальный сценарий, обслуживать нейтрально:
метка «Срочно», причина «Переезд» по желанию, пакет товаров, удобное
закрытие всех объявлений. Без маркетинга на трудностях.

## 10. Сообщество и офлайн

- **Антикафе — «рекомендуемое место встречи и проверки оборудования»**
  (не «безопасная сделка»): нейтральная точка, стол, электричество, люди,
  кофе. НЕ делать на первом этапе: хранение, передачу денег, подтверждение
  исправности, арбитраж, комиссию за встречу. Эволюция потом: точка встречи
  → базовая проверка → приглашённый техник → платная диагностика → лишь
  затем элементы защищённой сделки. Часы антикафе — ресурс отдельного
  партнёра, только по соглашению с ним.
- **Джемы и встречи** — двигатель ликвидности: знакомства → группы →
  вакансии → оборудование → записи.
- **Неденежный маркетинговый бюджет**: свободные часы студии, партнёрские
  бонусы, промокоды, площадки, экспертное время, охват. Считать по
  предельной себестоимости + занятости владельца, не по рыночной цене.
  Форматы наград — только с объективными условиями (розыгрыш по понятным
  критериям, открытый тестовый день для новых групп), без субъективных
  «конкурсов лучшего объявления».
- **Консьерж (ручное сведение)** — пилот в жёстких лимитах: срочные
  вакансии, дефицитные категории, несколько случаев в неделю, только
  согласившимся на персональные предложения; контакты без согласия не
  передаются (сначала ссылка на объявление). Будущий дорогой продукт
  «подбор кандидатов под задачу» — оценивать в доходе на час владельца.

## 11. Сербский язык — второй этаж (постепенно)

Этап 1: добровольное дублирование карточек на сербском.
Этап 2: сербоязычная публичная версия просмотра (барахолка, афиша,
организации). Этап 3: двуязычная публикация и поиск. Этап 4: совместные
мероприятия, мост между сценами. Автоперевод — с проверкой автором
(условия продажи, оплата, характеристики, даты). Гранты и пресса —
возможность, не финансовый план. Схема i18n в БД готова (`text_sr` рядом
с `text_en`).
