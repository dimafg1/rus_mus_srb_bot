## Правила внедрения

1. Ничего не удалять вслепую: у категорий есть привязанные объявления
   (65 шт.) — сначала карта переносов старая→новая, потом правки.
2. Поля наследуются по parent_id: общие поля вешать на корень раздела
   или родителя ветки, не дублировать по листьям.
3. Прежде чем менять БД — свежий бэкап: `python scripts/backup_db.py`.
4. UX: мастер подачи спрашивает только базу; flex-поля предлагаются после
   публикации, опциональны — количество полей не ограничено.
